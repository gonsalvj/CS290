Notes on Service Configuration

Configuring a WCF service depends on the requirement of the service, including where it will be hosted, how it will be consumed, what security is required, underlying protocols being used, and behaviours. These advanced topics are beyond the scope of this guide.

Confugration can be done within the web.config/app.config but it can also be implemented programmatically. Considerations when deploying a WCF service include security mode (message vs. transport), authentication (credential type - windows, username, certificate etc), and binding type.
