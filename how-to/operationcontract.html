Service and Operation Contracts

The next step in implementing a WCF service is to define the service and operation contracts. Similar to defining the data contract, this is accomplished by decorating our entities with specific attributes, namely ServiceContractAttribute and OperationContractAttribute.

The service contract essentially exposes a number of operations or 'calls' that will be available to a client. This is usually done via an interface but can be done using a regular class.

In order for the client to be able to make calls to specific operations within the service contract, method signatures within the interface must also be decorated by an attribute, [Operation Contract]. In this way, the client is not concerned with the implementation of the service, they only need to know what operations/calls are available.

Below is a sample Service contract with Operation contracts for a weather service.


[Service Contract]
public interface IForecast {

[OperationContract]
int getForecastInCelcius(string city);

[OperationContract]
int getForecastInFarenheit(string city);

[OperationContract]
int getProbabilityofPreciptation(string city);
}
