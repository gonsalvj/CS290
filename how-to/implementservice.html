Implementing Your Service

The service implementation is like any other class implementing some functionality. The key to the service however, is the the class must implement the interface defined as the Service Contract, as well as all Operation Contracts listed within the Service Contract. Note, the class and its methods and properties are not decorated with attributes from WCF, as this class is not exposed to the client. When the client does make a call from this service, these will be the methods that are invoked on the service side.


public class Forecast : IForecast {

    int getForecastInCelcius(string city) {
        ...
        return celcius;
    }

    int getForecastInFarenheit(string city) {
        ...
        return farenheight;
    }

    int getProbabilityofPreciptation(string city) {
        ...
        return pop;
    }
}
